# l10x Post-run: lets-develop-feature

## Release run lock on completion

When the change-code goal completes, release the workspace lock to unblock subsequent pipeline stages:

```bash
l10x run release $RUN_ID
```

If the goal exits with code 4 (workspace locked), release the oldest lock first:

```bash
l10x run list
l10x run release $OLDEST_RUN_ID
```

The `run_id` from this stage must be retained and passed to `lets-verify-change` — both stages share a run directory identified by this ID.
